# Talk Templates

> You (the CEO) don't need to be technical. Just use these templates.
> They've been refined through real conversations with the COO.

---

## 🚀 Starting a New Project

```
I want to build a [product type] for [target audience].

Examples:
- "I want to build a knowledge marketplace for developers."
- "I want to build a habit tracking app for students."
- "I want to build a invoice generator for freelancers."
```

## 🔧 Adding a Feature

```
Add [feature] to [project name].

Examples:
- "Add user login to the training platform."
- "Add a dashboard that shows daily revenue."
- "Add email notifications when someone purchases."
```

## 🐛 Reporting a Bug

```
There's a bug: [describe what's wrong in plain language].

Examples:
- "When I click checkout, nothing happens."
- "The login page shows an error after I enter my password."
- "Numbers on the dashboard don't add up."
```

## 📋 Asking for Progress

```
What's the status?

Examples:
- "What's the status?" (general check)
- "How's the payment feature going?"
- "Are we ready to deploy?"
```

## ✅ Approving Work

```
Looks good, proceed.
```
or
```
Not quite right. [describe what to change]
```

## 🎯 Setting Priorities

```
[This feature] is more important than [that feature].

Examples:
- "Payment is more important than the dashboard right now."
- "Make sure mobile works first, then desktop."
- "Speed matters more than fancy animations."
```

## 🔄 Asking for Changes

```
Change [what] to [new version].

Examples:
- "Change the color scheme to dark mode."
- "Change the price from $9 to $19."
- "Change the signup flow to just email, no password."
```

## 💬 When You Don't Know What to Say

```
I want [outcome], you figure out how.

Examples:
- "I want people to be able to pay with WeChat, you figure out how."
- "I want the site to load faster, you figure out how."
- "I want it to look professional, you figure out how."
```

---

## 💡 Pro Tips

1. **One thing at a time.** Don't ask for 5 features in one message. The COO works best with focused requests.
2. **Use plain language.** Don't try to sound technical. "Make the button blue" > "Update the CSS color property."
3. **Be specific about what you SEE, not what you THINK the fix is.** "The page is blank" > "Maybe the API is returning null."
4. **Trust the COO.** If you're not sure, say "you figure out how." That's literally the COO's job.
5. **Never say "make sure it works."** It's the most expensive sentence in software — it guarantees bugs. Just describe what "works" means to you, or let the COO figure it out.
