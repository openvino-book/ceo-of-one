## 🎯 你会学到什么

部署是一个商业决策，不仅仅是技术决策。

前面的章节里，CEO 说一句话，COO 全搞定。这一次，CEO 必须亲自注册账号、点击对话框、做配置选择。因为部署跨过了一条线——把你的项目从私有变成公开。一旦上线，任何有 URL 的人都能看到。这不只是技术里程碑，而是有真实后果的商业决策。

你会学到：
- 为什么 COO 准备，但 CEO 部署
- 哪一个设置毁了 90% 的首次部署
- 为什么数据在生产环境会消失（以及为什么这是好事）
- 从"本地 demo"到"真实产品"的心理转变

## 🗣️ CEO 说了什么

"先部署到vercel"

一句话。但这句话背后，CEO 做的工作比前面任何一章都多：

1. **注册 Vercel 账号** ——选择用 GitHub 登录
2. **导入仓库** ——从列表里选对仓库
3. **把 Root Directory 设为 "platform"** ——大多数人会漏掉的关键步骤
4. **验证部署** ——在浏览器里打开线上 URL，检查端点是否正常

CEO 没有写代码。但他们做了决策：用哪个平台、用哪种认证方式、怎么配置项目。在真实的一人公司里，你也要做同样的决策。没人会替你登录 Vercel。

这一章，"一人公司"不再是个比喻，而是变成了现实。你的代码在互联网上了。另一个国家的陌生人现在就能访问你的 URL。一切都不一样了。

## 🔍 做了什么

COO 做了四项部署准备：

**`.env.example`** ——添加了 `JWT_SECRET` 占位符。部署时需要告诉托管平台你的环境变量，这个文件就是模板。

**`next.config.js`** ——添加了 `output: 'standalone'`。告诉 Next.js 构建一个自包含的服务端包，针对 Vercel 等 serverless 平台优化。

**`src/app/api/health/route.ts`** ——添加了 `version` 字段。很小的改动，但很有用：在多个环境部署时，检查 `/api/health` 就知道哪个版本在跑。

**`vercel.json`** ——从零创建。告诉 Vercel 如何处理项目：路由规则、函数配置、框架检测。没有它，Vercel 会猜——而猜测可能是错的。

全部 94 个测试通过。构建零错误。COO 的目标是确保 CEO 点击"Deploy"时不会出问题。

确实没出问题。

## 🏗️ Root Directory ——部署的头号陷阱

这个问题需要单独一节，因为它是首次部署失败的最常见原因。

我们的项目是 monorepo。仓库根目录包含 `platform/`、`docs/` 等文件夹。Next.js 应用在 `platform/` 里面。Vercel 导入仓库时，默认从根目录构建。但根目录没有 `package.json`——它在 `platform/` 里面。所以构建失败，报一个关于缺少依赖的莫名错误。

解决办法就一个设置：**Root Directory = "platform"**。

这告诉 Vercel 去 `platform/` 文件夹里找应用。它能找到 `package.json`、`next.config.js`、`src/` 目录——它需要的一切。

如果你跟着做但部署失败了，先检查这个设置。90% 的"Vercel 构建不了我的项目"问题，靠这一行就解决了。

## 💡 空数据——不是 Bug，是一课

部署后，CEO 打开 `/api/courses`，看到：

```json
{"success": true, "data": []}
```

空的。没有课程。页面加载了，但什么也没有。

这不是 bug。这正好是应该发生的事。

我们的数据全部存在内存里。每次服务器启动（Vercel 每次部署都会重新启动），内存存储重置为空。在你本地机器上，你看不到这个问题，因为开发服务器一直在跑。在生产环境里，服务器频繁启停，问题就显而易见了。

这一刻，你理解了为什么每个真实应用都需要数据库。不是因为别人告诉你，而是因为你的线上站点证明了这一点。你部署了代码，代码运行完美——但没有数据，因为你从来没有保存过任何东西。

第九章就是解决这个问题的。你要加数据库、持久化存储课程、搭建管理仪表盘。但动力来自这一章——来自看到空白页面，理解了**为什么**。

## 🎓 本章收获

到目前为止，这个项目只是你讲给自己听的故事。"我在做一个课程平台。"但除了你，没人看得到。

今天，情况变了。首页上线了。API 端点正常响应。一个 URL 存在了，互联网上的任何人都能访问。这不是小事。

技术工作很少——四个配置文件，零 bug，现有测试全部通过。真正的工作是 CEO 的：注册账号、做部署决策、验证结果。在一人公司里，你跳不过这些步骤。你就是那个点击"Deploy"的人。

而空数据呢？那是一份礼物。它给你紧迫感。第九章不再是抽象的"最佳实践"——它是挡在你的线上站点和用户可用的真实产品之间的东西。

**一人公司原则：** 部署是你的项目变成现实的时刻。认真准备你的代码，但记住——点击"Deploy"的那个人，比他们部署的代码更重要。
