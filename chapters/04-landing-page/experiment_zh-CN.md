# 第四章实验：搭建真实的训练营官网首页

## 目标

这不是对比实验，而是一次**真实的产品构建**——完整演示 CEO of One 工作流：CEO 说一句话 → COO 拆解 → 工程师执行 → 产出生产级交付物。

## 配置

| 项目 | 详情 |
|---|---|
| **CEO 输入** | 一句话："帮我搭建 CEO of One 训练营的官网首页。" |
| **COO（OpenClaw）** | 拆解为技术栈、设计规范、页面分区、验收标准 |
| **工程师（Claude Code）** | 通过 acpx 一次性完成全部构建 |
| **技术栈** | Next.js 14 + TypeScript + Tailwind CSS |
| **耗时** | 约 10 分钟 |

## 过程

### 第一步：CEO — 一句话

CEO 用一句中文描述了需求，没有设计稿，没有功能清单，没有后续补充。

### 第二步：COO — 任务拆解

OpenClaw 将一句话需求拆解为完整构建方案：

- **技术栈：** Next.js 14、TypeScript、Tailwind CSS
- **9 个页面区块：** 导航栏、首屏、痛点、方案、课程大纲、社会认证、价格、FAQ、页脚
- **设计规范：** 暗色主题（#0a0a0a 底色、#1a1a2e 卡片、#6366f1 强调色）、响应式布局
- **10 条验收标准：** 构建通过、lint 无警告、无 TODO/FIXME/console.log、全部区块实现、响应式、FAQ 手风琴交互、吸顶导航等

### 第三步：工程师 — 构建

Claude Code 收到方案后一次性完成全部页面开发——两个源文件共 389 行。

## 结果

| 验收项 | 状态 |
|---|---|
| `npm run build` | ✅ 零错误 |
| `npm run lint` | ✅ 零警告 |
| 无 TODO/FIXME/console.log | ✅ 代码整洁 |
| 9 个页面区块全部实现 | ✅ |
| 暗色主题 | ✅ #0a0a0a / #1a1a2e / #6366f1 |
| 响应式（移动端 + 桌面端） | ✅ |
| FAQ 手风琴交互 | ✅ 点击展开/收起 |
| 吸顶导航栏 | ✅ |

**交付物：** 2 个源文件（`layout.tsx` 362 行 + `page.tsx` 27 行），一次通过，生产可用。

## 关键洞察

**1. 一句话 → 生产级页面。** CEO 没有提供任何技术细节。COO 的拆解将模糊需求转化为精确的构建方案——这是 CEO of One 方法的核心价值。

**2. 瓶颈在拆解，不在编码。** Claude Code 能在几分钟内完成开发，正是因为需求已经没有歧义。COO 的工作（区块定义、设计规范、验收标准）才是关键。

**3. 验收标准防止质量滑坡。** 如果没有"无 TODO、无 console.log、构建和 lint 必须通过"这些约束，交付物很可能带着开发残留物。这些标准定义成本为零，却省掉了一轮 review。

**4. 自指证明。** 你在本章末尾看到的落地页，就是这次实验的真实产出。读者看到的是本书方法构建的真实产物——不是 mockup，不是演示。

**5. 10 分钟、1 句话、389 行。** 这是 ROI 数字。人类团队需要启动会、设计稿、前端开发、review、修改——跨越数天。AI 团队在喝杯咖啡的时间里完成了。

## 这证明了什么

CEO of One 模型不是理论，而是可复现的流程：
1. 你说你想要什么（1 句话）。
2. 你的 AI COO 把它变成计划（拆解）。
3. 你的 AI 工程师把它造出来（执行）。
4. 自动化检查保证质量（验收标准）。

这套方法不局限于落地页——同样的流程适用于后端服务、移动应用、文档编写等。但这次实验证明了基线：**一个拥有 AI 团队成员的独立创业者，能够以真实速度交付真实产品。**
