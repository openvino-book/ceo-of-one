# 第 2 章实验：指令精确度与贪吃蛇游戏

## 目标

通过用三种不同详细程度的指令构建同一个贪吃蛇游戏，然后逐项对比结果与规格的匹配度，衡量指令精确度如何影响 AI 输出的保真度。

## 实验设置

| 项目 | 详情 |
|---|---|
| **任务** | 构建一个网页版贪吃蛇游戏（单 `index.html` 文件） |
| **工具** | acpx + Claude Code + COO 灵魂模板 |
| **变量** | 指令精确度（模糊 / 中等 / 精确） |
| **评估方式** | 输出是否符合要求？逐项功能对比。 |

## 实验过程

### 步骤 1：版本 A — 模糊指令

```
Build a Snake game.
```

Claude Code 产出了一个可玩的贪吃蛇游戏。能用。但：
- 蛇是蓝色的，不是绿色的（我们没指定颜色）
- 游戏速度约 100ms（太快，我们没指定）
- 没有重新开始按钮（我们没要求）
- 画布尺寸不标准

**结论：** 功能可用。但不是我们想要的。只是我们没说出我们想要什么。

### 步骤 2：版本 B — 中等指令

```
Build a Snake game. Single HTML file. The snake grows when eating food.
Game over when hitting walls or itself. Include arrow key controls.
```

Claude Code 添加了要求的功能，还发挥了创意——给蛇加了眼睛、渐变身体和脉动的食物动画。

**结论：** 更好。核心机制匹配。额外功能在原型阶段是惊喜，但如果有特定视觉要求就不想要了。

### 步骤 3：版本 C — 精确指令

```
Build a web-based Snake game. All code in a single index.html file.
- Canvas 400x400, dark background #1a1a2e, green snake #00ff41, red food #ff0044
- Snake starts center, moving right, length 3
- Arrow keys, prevent 180-degree reversal
- +10 points per food, grow on eat
- Game over on wall/self hit, score display, restart button
- 150ms game loop, pure HTML/CSS/JS
```

Claude Code 产出了完全符合规格的结果。每个颜色、每个尺寸、每个行为都匹配。

**结论：** 精确匹配。没有意外。这就是生产环境需要的。

### 步骤 4：逐项功能对比

## 实验结果

| 功能 | 模糊 | 中等 | 精确 |
|---|---|---|---|
| 可玩的贪吃蛇 | ✅ | ✅ | ✅ |
| 单 HTML 文件 | ✅ | ✅ | ✅ |
| 正确画布尺寸 | ❌ | ❌ | ✅ 400×400 |
| 深色背景 | ❌ | ❌ | ✅ #1a1a2e |
| 绿色蛇 | ❌（蓝色） | ❌（随机） | ✅ #00ff41 |
| 红色食物 | ❌ | ✅ | ✅ #ff0044 |
| 方向键控制 | ✅ | ✅ | ✅ |
| 防止 180° 掉头 | ❌ | ✅ | ✅ |
| 分数显示 | ❌ | ✅ | ✅ |
| 重新开始按钮 | ❌ | ❌ | ✅ |
| 游戏循环速度 | ❌（~100ms） | ❌（~100ms） | ✅ 150ms |
| 规格匹配率 | ~30% | ~60% | **100%** |

## 关键发现

1. **精确不会更贵——反而更便宜。** 精确版（版本 C）实际上是最便宜的一次运行，因为 Claude Code 不需要浪费 token 去猜测或迭代模糊的需求。模糊指令迫使 AI 做假设，错误的假设要花 token 来修正。

2. **方法论流程仍然不完美。** 和第 1 章相同的问题：OpenClaw（COO）直接运行了对比，而不是遵循 CEO→COO→工程师 的流水线。在成熟的流程中，CEO 说"对比这三种提示方案"，COO 应该独立协调三次工程师运行。我们在这里做得更临时。

3. **AI 在你模糊时发挥创意，不是在你精确时。** 版本 B 的蛇眼睛和渐变身体是"AI 的创意"——原型阶段有趣，生产阶段有问题。精确消除了不想要的创意，但没有消除能力。

4. **精确度公式很简单：做什么 + 怎么工作 + 看起来什么样。** 三个类别几乎覆盖了 CEO 需要指定的一切。你不需要写代码，你只需要描述你脑海中的画面。

## 这证明了什么

指令精确度与输出保真度呈直接线性关系。模糊和精确之间没有"甜蜜点"——对于生产环境，精确总是胜出。精确度公式（做什么 + 怎么工作 + 看起来什么样）为非技术 CEO 提供了一个在不写代码的情况下给出清晰指令的框架。

## 沉淀知识

- B：模糊指令产出功能可用但不可预测的输出。规格匹配率约 30%。
- B：精确指令产出精确匹配。规格匹配率 100%。
- B：精确度公式：做什么 + 怎么工作 + 看起来什么样。
- B：精确实际上降低成本——减少假设-迭代循环。
- O：AI"创意"（版本 B 的蛇眼睛）在原型阶段是特性，在生产阶段是 bug。
- O：COO 应该独立协调 A/B 测试运行（方法论修复在后续章节中应用）。
