# 实验：第 2 章验证 —— 指令精确度对比

> **日期：** 2026-03-20
> **目标：** 对比模糊、中等、精确三种指令对同一任务的结果差异
> **状态：** ✅ 已完成

## 假设

越精确的指令，产出越接近你的想象；越模糊的指令，产出越"能用但不是你想要的"。

## 实验设计

**任务：** 做一个贪吃蛇游戏。

**对照组 A（模糊）：** `"Make a snake game"`
**对照组 B（中等）：** `"Build a web-based Snake game using HTML, CSS, and JavaScript. The snake grows when it eats food. The game ends when the snake hits a wall or itself."`
**对照组 C（精确）：** 完整规格说明（画布大小、颜色、控制方式、游戏速度、文件结构、验证步骤）。

## 结果

| 指标 | A（模糊） | B（中等） | C（精确） |
|------|----------|----------|----------|
| **文件大小** | 5,457 字节 / 182 行 | 12,440 字节 / 388 行 | 8,997 字节 / 309 行 |
| **画布** | ✅ | ✅ | ✅ |
| **Game Over 画面** | ✅ | ✅ | ✅ |
| **深色背景 (#1a1a2e)** | ✅ | ✅ | ✅ |
| **分数显示** | ✅ | ✅ | ✅ |
| **重新开始按钮** | ✅ | ✅ | ✅ |
| **绿色蛇 (#00ff41)** | ❌ | ❌ | ✅ |
| **180 度反转防止** | ❌ | ✅ | ✅ |
| **150ms 游戏循环** | ❌ | ❌ | ✅ |
| **创建后验证文件** | ❌ | ✅ | ✅ |
| **额外功能** | — | 蛇的眼睛、渐变身体、最高分、暂停 | — |

## 关键发现

### 对照组 A：能用
- 做出了一个能玩的贪吃蛇
- 缺少：精确颜色、180 度反转防止、游戏速度控制
- AI 自作主张选了文件名

### 对照组 B：超预期
- 代码最多（388 行），额外加了蛇眼睛、渐变身体、最高分保存、暂停功能
- 缺少：精确颜色 (#00ff41)、精确速度 (150ms)
- AI 发挥创意加了没要求的功能

### 对照组 C：精准匹配
- 每一条规格都精确满足
- 颜色对了、速度对了、控制方式对了
- 连文件名都按要求叫 `index.html`
- 还跑了 `ls -la` 验证文件

### 精确度悖论

有趣的是，B 比 C 产出了更多代码。因为精确指令约束了输出，而中等指令给了 AI 发挥空间。好坏取决于目标：

| 目标 | 最佳方式 |
|------|---------|
| "做点酷的东西，给我惊喜" | B（中等） |
| "就按我说的做，不多不少" | C（精确） |
| "能玩就行" | A（模糊） |

## 结论

**假设成立，但有一个细微差别。**

- **精确 = 控制。** 想要精确的颜色、速度和行为，必须明确指定。
- **模糊 = 委托。** AI 会替你做决定。有时这样挺好，有时不行。
- **最佳实践：** 创意任务用中等精确度，有明确愿景时用精确指令。

**对一人 CEO 来说：** 你通常想要精确。你是产品愿景家——AI 应该执行你的愿景，而不是自由发挥。
